# MurderTransformComponentsLookup

**Namespace:** Bang \
**Assembly:** Murder.dll

```csharp
public class MurderTransformComponentsLookup : MurderComponentsLookup
```

Additional lookup class on top of the Bang generated one. Needed for adding
            IMurderTransformComponent to the relative component lookup table with the correct id.

This is a second, smaller class generated by `Bang.Generator` alongside [MurderComponentsLookup](../Bang/MurderComponentsLookup.html), and is
the one actually instantiated and used at runtime (it is the most-derived `ComponentsLookup`
implementation found when [World.FindLookupImplementation](../Bang/World.html#findlookupimplementation) scans loaded assemblies). It exists because
`Bang.Generator` treats transform components specially: every project's transform component shares
the single, engine-wide `BangComponentTypes.Transform` id instead of getting its own per-project id
(see the `IsTransformComponent` handling in `Templates.LookupImplementation`), so
[MurderComponentsLookup](../Bang/MurderComponentsLookup.html) alone cannot register `IMurderTransformComponent` into `RelativeComponents` with the
correct shared id. `MurderTransformComponentsLookup` patches that in, and exposes
`MurderTransformNextLookupId` as the next free id after everything both classes have reserved.

As with [MurderComponentsLookup](../Bang/MurderComponentsLookup.html), you should not construct or reference this class directly, and it
should never be hand-edited — it is regenerated on every build. Use [World](../Bang/World.html) and [ComponentsLookup.Id(Type)](../Bang/ComponentsLookup.html#idtype) to
resolve component ids instead of touching this class.
